# SUJI Aesthetic Direction

> 岁吉视觉方向备忘：从中式美学参考图中抽取可产品化的底层元素。  
> 目标不是做“国风模板”，而是建立一套能落到移动 App、命理可视化、AI 问道和 self-care 仪式里的当代东方触感系统。

---

## 0. 一句话方向

**纸为底，墨为骨，朱砂为魂；以留白养气，以器物入世，以手作痕迹抵抗数字感。**

更产品化地说：

> **Neo-Chinese Tactile Divination**  
> 当代东方触感命理美学。

SUJI 的中式感不应该来自龙、凤、祥云、宫廷金、庙宇符咒这些高噪声符号，而应该来自更底层的视觉语法：纸、墨、朱砂、留白、竖排、印章、器物、手作 texture、古今混排。

---

## 1. 核心判断：什么让它“中式”

这批参考图里的“中式”不是单一符号，而是一组共同特征：

1. **承载物先行** — 画面先像纸，再承载文字和图形。
2. **墨色有生命** — 黑不是纯黑，而是湿墨、干墨、拓印黑、木刻黑。
3. **朱砂只点睛** — 红色小而重，像印章、落款、时间戳。
4. **留白承载气韵** — 空白不是没内容，而是让主体有呼吸。
5. **文字具有空间性** — 竖排、手写、古籍栏线、稿纸格线都在建立东方阅读气质。
6. **图形来自日常器物** — 茶、碗、壶、窗、纸、山、水、叶、桌，而不是廉价玄学符号。
7. **材质保留不完美** — 木刻刮痕、水彩渗色、纸纤维、印章缺口、毛边、墨迹浓淡。
8. **古典情绪 + 现代网格** — 情绪可以古典，但信息组织必须当代、清晰、可用。

## 3. 设计原则

### 3.1 Warm Paper First

所有界面先像纸，不像玻璃。

**Do**
- 使用暖白、米汤色、宣纸白、旧书页色、低饱和灰黄作为背景。
- 保留非常轻的纸纤维、噪点或水彩纸颗粒。
- 卡片可以像纸片、签纸、旧页、笺，而不是纯现代 SaaS card。

**Avoid**
- 纯白 `#FFFFFF` 大面积铺底。
- 玻璃拟态、荧光渐变、赛博紫蓝。
- 过重阴影导致“iOS 模板卡片感”。

**UI translation**
- 首页黄历像一张可撕的纸。
- AI 问道回复像一页小笺。
- 命盘不是仪表盘，而是一张被整理过的命理手稿。

---

### 3.2 Ink, Not Black

黑色要像墨，有浓淡、粗细和呼吸。

**Do**
- 使用墨黑、炭黑、烟灰黑，而不是纯黑。
- 关键标题可以带轻微笔触感、拓印感或手写感。
- 线条允许一点不规则：像手绘、木刻、版画，而不是机械矢量。

**Avoid**
- `#000000` + 系统字体的硬 UI。
- 大量等粗 stroke icon。
- 所有图形边缘都过于干净。

**UI translation**
- 十神 / 五行 / 奇门宫位里的符号可以像印刷在纸上。
- 分割线可以是淡墨线、稿纸线、折痕线。
- icon 系统可以参考木刻 pictogram，而不是通用 outline icon。

---

### 3.3 Cinnabar as Ritual

朱砂红不是 warning red，而是“落款红”。

**Do**
- 小面积使用在印章、关键状态、完成确认、日期、今日重点。
- 红色应偏朱砂、印泥、赭红，而不是 UI error red。
- 用红色制造仪式感，而不是制造焦虑。

**Avoid**
- 大面积中国红背景。
- 红金配色堆砌。
- 把朱砂红用成普通 CTA 色。

**UI translation**
- “今日已阅”用小印章，而不是绿色 check。
- “吉 / 平 / 慎”可以做成不同形态的小印。
- AI 问道完成后落一枚“已问”印。

---

### 3.4 Whitespace Carries Qi

留白是内容的一部分。

**Do**
- 用大留白承载情绪，特别是每日运势、呼吸、签文、启动页。
- 信息密集页也要有“气口”：分组、栏线、空隙、视觉停顿。
- 允许一张卡片只表达一个主旨。

**Avoid**
- Dashboard 化：一屏塞满数字、icon、tag、进度条。
- 每个空白都用解释文字填满。
- 为了显得“高级”而加无意义小字。

**UI translation**
- 每日卡片优先一句话 + 一个主符号 + 一个小印。
- 奇门 / 八字 / 六爻等复杂结果要分层展开，先给用户情绪锚点，再给细节。

---

### 3.5 Every Result Needs a Seal

关键结果需要一个“收束动作”。

印章在中式系统里不只是装饰，它代表认证、落款、私人性、时间戳、完成感。

**Potential seal system**
- `岁吉` 主印
- `今日` 日签印
- `已阅` 完成印
- `已问` AI 问道印
- `吉 / 平 / 慎` 状态印
- 用户自己的 `命印`
- 节气印、五行印、宫位印

**UI translation**
- 用户完成一次问道后，回复尾部落印。
- 每日黄历撕下后出现“今日已阅”。
- 命盘生成完成后有一枚私人命印，而不是普通 success toast。

---

### 3.6 Daily Objects Over Mystic Clichés

中式命理不一定要“玄”。SUJI 更适合从人间生活里长出来。

**Preferred motifs**
- 纸、笺、册页、稿纸、折痕
- 茶、碗、壶、烟、炉
- 山、水、叶、风、雨、节气
- 窗、桌、格线、灯、帘
- 印、签、签筒、刻痕

**Avoid / use very carefully**
- 龙凤祥云
- 金色宫廷纹样
- 八卦符号满屏铺陈
- 仙侠感、庙宇感、符咒感
- 过度神秘化的 tarot / occult visual language

**Product stance**
> 岁吉不是恐吓式算命，也不是古风玄学工具。  
> 它是一个把命理、情绪和日常仪式轻轻放在一起的 self-care companion。

---

### 3.7 Ancient Mood, Modern Grid

情绪古典，结构现代。

**Do**
- 古籍竖排可以作为标题或诗句层，不作为全局阅读方式。
- 现代网格负责信息清晰度。
- 中英文 / 古今混排可以让产品更当代。
- 信息密度要被产品结构控制，而不是被视觉情绪淹没。

**Avoid**
- 为了古典而牺牲可读性。
- 大段竖排正文。
- 把 App 做成复古海报集合。

**UI translation**
- 竖排用于 ceremonial layer：签文、标题、命格短句、节气名。
- 正文解释仍使用横排现代排版。
- 命盘信息使用清晰分区，情绪层覆盖在外。

## 4. Texture 语言

### 推荐 texture

- 宣纸纤维
- 米纸颗粒
- 旧纸黄斑
- 水彩纸冷压纹理
- 印章破损边缘
- 木刻 / 版画刮痕
- 手写墨水浓淡
- 毛毡 / 织物绒边
- 青花蓝单色纸纤维

### Texture 使用规则

1. **底纹轻，局部重。** 背景 texture 不抢内容；印章、插画、标题可以更重。
2. **同屏不超过 2 种材质主导。** 比如纸纹 + 印章，不要纸纹 + 水彩 + 木刻 + 毛毡全上。
3. **texture 为情绪服务，不为装饰服务。** 如果用户注意到“好多纹理”，说明过了。
4. **移动端优先清晰度。** 小字号正文不要叠太重的颗粒。

## 5. 色彩方向

### Foundation

- 宣纸白：温暖、轻微发黄，不纯白。
- 米汤色：用于主背景或卡片底。
- 旧书页色：用于历史记录、日签、档案感页面。
- 淡灰绿 / 青瓷绿：用于节气、呼吸、自然感。
- 淡赭 / 土陶色：用于紫砂、烟火气、暖提示。

### Ink

- 墨黑：主文字。
- 炭灰：次级文字。
- 淡墨：分割线、辅助框线。

### Accent

- 朱砂红：仪式点睛。
- 印泥红：印章 / 落款。
- 青花蓝：概念性 key visual，不宜过度日常化。

### Avoid

- 高饱和中国红大面积使用。
- 金色大面积使用。
- 紫蓝荧光渐变。
- 情绪疗愈 App 常见的 generic pastel rainbow。

---

## 6. 字体与排版方向

### 中文

- 标题层：宋体 / 明朝体 / 仿碑帖字形 / 手写字形可以探索。
- 正文层：需要足够现代、清晰，不要为了中式牺牲可读性。
- 情绪层：可使用竖排、手写、签文式短句。

### 英文 / 数字

- 小号英文可以像展览标签、香水标签、标本卡。
- 数字可带档案编号感：节气编号、日签编号、排盘时间。
- 英文不是装饰垃圾文本，必须有信息意义。

### 排版手法

- 竖排标题 + 横排正文。
- 古籍栏线 / 稿纸格线作为结构提示。
- 小字落款 / 日期 / 坐标制造档案感。
- 大标题少而重，小字轻而稳。

---

## 7. Icon / Illustration 系统

### Icon 方向

优先探索三类：

1. **木刻 pictogram**
   - 适合：五行、节气、命理概念、导航辅助图形。
   - 特点：黑色剪影、刮痕、粗糙边缘、像民艺标本。

2. **篆刻 seal icon**
   - 适合：状态、等级、完成、认证、用户命印。
   - 特点：红黑双色、方圆容器、阴刻 / 阳刻。

3. **器物极简线描**
   - 适合：空状态、呼吸、ambiance、首页装饰。
   - 特点：少线条、大留白、一物一境。

### Illustration 方向

- 水彩文人静物：茶、壶、烟、纸、窗。
- 毛毡节气拼贴：温暖、可爱但不幼稚。
- 单色剪纸意象：碗中山水、窗中月、纸上映影。

## 8. 具体落地到 SUJI 模块

### 首页 / 手撕黄历

- 视觉隐喻：纸页、日签、旧历、可撕边。
- 主结构：大留白 + 今日一句 + 小印 + 少量细节。
- Texture：纸纹、撕纸边、淡墨线。
- Avoid：一屏塞满宜忌、分数、进度条。

### AI 问道 / Chat

- 视觉隐喻：书信、小笺、问答手稿。
- AI 回复可以有“摘句”层：一句竖排或手写式核心建议。
- 长回答需要现代阅读结构：分段、缩进、轻分隔。
- 结束处可落“已问”印。

### 八字 / 命盘 / 奇门

- 视觉隐喻：整理过的命理手稿，而不是数据仪表盘。
- 第一层给用户情绪锚点：今天该怎么看自己。
- 第二层才展开技术细节：十神、五行、宫位、用神。
- 宫位 / 卡片可参考古籍栏线与现代 grid 结合。
- 印章可用于标记关键宫、用神、吉凶状态。

### Ambiance / 呼吸 / 正念

- 视觉隐喻：茶烟、窗影、山水、雨、节气。
- 更适合东方极简和水彩文人静物。
- 动效应慢、轻、低饱和：烟气、纸张微动、墨晕扩散。

### Profile / 用户命印

- 可以生成或选择一个“命印”作为私人身份符号。
- Profile 不是设置页堆叠，而像一本私人命理册页。
- 用户生日 / 时辰 / 城市可以用小字落款方式呈现。

## 9. 反模式清单

这些会把 SUJI 拉向廉价国风或 generic AI design：

- 中国红 + 金色 + 祥云满屏。
- 龙凤、宫殿、仙侠人物、庙宇符咒直接堆叠。
- 过度神秘化、恐吓式算命视觉。
- 玻璃拟态卡片 + 紫蓝渐变。
- Emoji 当 icon。
- Every card 都是圆角白卡 + 左侧彩色 border。
- 无意义的英文小字 / pseudo-Japanese decoration。
- 复杂命理信息未经分层直接塞满一屏。
- 纹理过多导致移动端脏、糊、廉价。
