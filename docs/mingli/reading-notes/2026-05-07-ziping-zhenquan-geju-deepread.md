## 一、论正官 / 论正官取运（成败救应）

### 短摘录

1. `存其喜而去其忌则贵。`
2. `正官露而不可逢合，不可杂煞，不可重官。`
3. `以财印并透者论之，两不相碍，其贵也大。`
4. `以印能护官，亦能泄官，而财生官也。`
5. `刑冲破害，以下犯上，乌乎可乎？`

### 算法可用规则

| 类型 | 规则 |
|---|---|
| 取格 | 月令本气透干 = 正官 → 用神 = 正官 |
| 相神（首选） | 财生官、印护官；财印并透且不相碍 = 大贵 |
| 成格条件 | 财印齐透不相碍；身能任官；官星清纯不杂煞 |
| 破格条件 | 官被合（合官）；官杂七煞；重官；刑冲破害月令官星 |
| 救应路径 | 杂煞被合去 → 取清；重官被合去一 → 取清；伤官见印护官 |
| 忌神 | 伤官（直接克官）、合神（贪合忘官） |

### 工程落点

- `determineGeJu` 当前"月支主气十神 == 正官"即定格——**正确但不充分**。需追加：
  - `chengBai` 判定：财印是否齐透 / 是否被合 / 是否杂煞 / 是否重官
  - `xiangShen` 标记：哪个柱的财/印承担相神职能
  - `jiuYing` 路径：如杂煞场景下"合煞留官"的检测
- 无新数字阈值，全部布尔/枚举判断。

## 二、论财 / 论财取运（成败救应）

### 短摘录

1. `身强而不透伤官，不混七煞，贵格也。`
2. `财喜根深，不宜太露，然透一位以清用，格所最喜。`
3. `有财用伤官者，财不甚旺而比强，略露一位伤官以化之。`
4. `孤财不贵，佩印帮身，即印取贵。`
5. `贪财破印，贫贱之局也。`
6. `有财带七煞者，或合煞存财，或制煞生财，皆贵格也。`

### 算法可用规则

| 类型 | 规则 |
|---|---|
| 取格 | 月令本气透干 = 正财/偏财 → 用神 = 财 |
| 相神（首选） | 食神生财（最纯）、伤官生财（次）、印帮身（身弱时） |
| 成格条件 | 财根深透一位 + 身能任财 + 不混七煞 + 不透伤官（身强时） |
| 破格条件 | 比劫夺财（无食伤化、无官杀制）、贪财破印（财克印致身无依）、孤财无生 |
| 救应路径 | 比劫夺财 → 食神化比劫生财 / 官星制比劫；财带七煞 → 合煞或制煞 |
| 身弱配合 | 财多身弱 → 必佩印帮身（"印取贵"），印不可被财破 |

### 工程落点

- 财格是当前 `determineGeJu` 处理最弱的——只标格名，不评 `chengBai`、不识"贪财破印"、不识"比劫夺财→食伤化"救应链。
- `xiangShen` 字段对财格特别有意义：`{type: 'food-stem', position: '月干'}` vs `{type: 'seal-stem', position: '年干'}` 决定走食伤生财还是印帮身路径。

## 三、论印绶 / 论印绶取运（成败救应）

### 短摘录

1. `印绶喜其生身，正偏同为美格。`
2. `身旺印强，不愁太过，只要官星清纯。`
3. `若印轻财重，又无劫财以救，则为贪财破印，贫贱之局也。`
4. `有印而用伤食者，身强印旺，恐其太过，泄身以为秀气。`
5. `用煞而有制，生身而有泄，不论身旺印重，皆为贵格。`
6. `印多而用财者，印重身强，透财以抑太过，权而用之，只要根深，无防财破。`

### 算法可用规则

| 类型 | 规则 |
|---|---|
| 取格 | 月令本气透干 = 正印/偏印 → 用神 = 印 |
| 相神（首选） | 官星生印（最贵）；身印俱旺时改用食伤泄秀 |
| 成格条件 | 印生身 + 官护印（清纯无杂） / 杀印相生（杀得制） / 印重时透食伤泄秀 / 印重时透财权用 |
| 破格条件 | 财破印（印轻财重无劫救）、伤官见官（虽印护官但易受伤）、印过重身浮 |
| 救应路径 | 财破印 → 比劫救印（劫财夺财存印）；印重身浮 → 食伤泄秀；印缺 → 官杀生印 |
| 反向规则 | "印重透财"是反规则——要求 `根深` + `身强` + `权用` 三条件齐全 |

### 工程落点

- 印格的"反规则"（印重透财、印重见食伤）说明`yongShen`选择不能机械化——必须根据日主+印的相对力量切换路径。
- 这呼应 `bazi.yongshen-priority.product-decision` claim：用神优先级是 product 决策，《子平真诠》给的是"看局势变化"而非固定排序。

## 四、论偏官（七杀）/ 论偏官取运（成败救应）

### 短摘录

1. `煞以攻身，似非美物，百大贵之格，多存七煞。`
2. `煞用食制者，上也，煞旺食强而身健，极为贵格。`
3. `煞用食制，不要露财透印，以财能转食生煞，而印能去食护煞也。`
4. `有七煞用印者，印能护煞，本非所宜，而印有情，便为贵格。`
5. `亦有煞重身轻，用食则身不能当，不若转而就印。`
6. `有煞而用财者，财以党煞，本非所喜。`
7. `或去官，或去煞，取清则贵。`

### 算法可用规则

| 类型 | 规则 |
|---|---|
| 取格 | 月令本气透干 = 七杀 → 用神 = 七杀 |
| 处理三路径 | **制**（食神制杀）：上格；**化**（印化杀）：印有情贵；**合**（阳刃合杀）：杀刃相济 |
| 路径选择 | 煞旺身健 → 用食制；煞重身轻 → 用印化（不可用食制，身不能当）；阳干日主 → 阳刃合杀 |
| 成格条件 | 食制 + 不露财透印；印化 + 印有情；合杀 + 阳刃专格 |
| 破格条件 | 食制路径下露财（财转食生煞）或透印（印去食护煞）；杀混正官（官煞混杂）； |
| 救应路径 | 官煞混杂 → 去官留煞 / 去煞留官（取清）；食制被印破 → 财去印 |
| 忌神 | 食制路径忌财、忌印；印化路径忌财（财坏印）；合杀路径忌冲（冲开合局） |

### 工程落点

- 七杀格的"三路径选择"是当前 `determineGeJu` 完全没有的能力。需要 `chengBai` 字段携带子状态：`{path: 'shi-zhi'|'yin-hua'|'ren-he', xiangShen: {...}}`
- "煞重身轻 → 转而就印"是 **强弱判断 ↔ 用神选择** 强耦合的典例——再次说明不能用单一阈值二元化。
- 当前 `getCongGe` 处理"从杀格"，但《子平真诠》正格七杀和从杀格是不同的判定路径（正格要身有根、要制化合；从杀要四柱无根、纯杀势力）。

## 综合：4 格的共同结构

| 共同字段 | 4 格表现 |
|---|---|
| `name` | 正官 / 正财|偏财 / 正印|偏印 / 七杀 |
| `yongShen` | 月令本气 / 透干十神 |
| `xiangShen` | 财生官、印护官 / 食神生财、印帮身 / 官生印、食伤泄秀 / 食制杀、印化杀、刃合杀 |
| `chengBai` | 'cheng' / 'po' / 'jiuying' |
| `chengBaiPath` | 各格 specific 子枚举（如七杀的 制/化/合） |
| `jiuYing[]` | 救应链：被破字 → 救字 → 路径名 |
| `jibie` | 上/中/下，依据：相神是否齐 + 是否清纯 + 救应是否完美 |
